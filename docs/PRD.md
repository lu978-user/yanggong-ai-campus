# 扬工智行 - AI 智慧校园公益服务助手 PRD

## 1. 项目定位

扬工智行是一款面向校园师生与访客的 AI 智慧校园公益服务助手。平台以 Dify Chatflow API 为智能编排后端，将校园导览、图书馆、食堂、成长规划、反诈宣传、心理关怀六类高频公益服务整合到一个现代化校园服务工作台中。

产品不定位为普通聊天机器人，而是“校园任务入口 + 场景智能体 + 会话历史”的服务平台。

## 2. 目标用户

- 新生：快速熟悉校园建筑、服务流程和学习资源。
- 在校学生：获得食堂、图书馆、成长规划、心理支持等日常帮助。
- 辅导员与校园服务人员：通过标准化 AI 入口提升公益宣传和服务触达。
- 访客：快速了解校园主要建筑和功能。

## 3. 核心目标

- 首页六宫格承载六类校园服务入口。
- 点击卡片进入对应场景聊天，并自动预填/发送问题。
- 聊天页支持 Dify Chatflow API 流式输出。
- 支持本地会话历史，用户可以查看、切换、继续历史对话。
- 支持深色模式与移动端适配。
- 整体视觉参考 Apple、ChatGPT、Notion、Vercel：简洁、高级、毛玻璃、低噪声、清晰层级。

## 5. 六类服务入口

| 服务 | 默认问题 | 核心能力 |
| --- | --- | --- |
| 校园导览 | 请介绍学校主要建筑和功能 | 建筑介绍、路线建议、服务点位 |
| 图书馆助手 | 请帮我了解图书馆开放时间、借阅规则和学习资源 | 借阅规则、资源检索、阅读推荐 |
| 食堂助手 | 请推荐今天适合学生的食堂窗口和营养搭配 | 窗口推荐、营养搭配、高峰提醒 |
| 成长规划 | 请根据大学生成长目标制定一份学习和职业规划 | 学习计划、技能路线、职业建议 |
| 反诈宣传 | 请介绍校园常见诈骗套路以及防范方法 | 风险识别、案例科普、应急指引 |
| 心理关怀 | 我最近有些压力，请给我一些温和的心理支持和校园求助建议 | 情绪支持、压力缓解、资源引导 |

## 6. 核心交互流程

### 场景入口流程

1. 用户在首页点击六宫格卡片。
2. 系统跳转到 `/chat?service=xxx&q=xxx`。
3. 聊天页识别服务场景和问题。
4. 自动创建会话或复用当前会话。
5. 自动发送问题。
6. Dify 通过流式接口返回回答。
7. 前端逐字/逐块渲染回答。
8. 会话写入本地历史。

### 自由提问流程

1. 用户在聊天页输入问题。
2. 前端携带当前服务场景、conversation_id、本地 session_id 调用后端接口。
3. 后端转发至 Dify Chatflow API。
4. 前端以流式方式展示回答。
5. 回答完成后更新会话历史。

## 7. 会话历史

本期采用浏览器 localStorage 存储：

- 会话标题：取第一条用户问题前 18 个汉字。
- 会话场景：保存 serviceId。
- 会话消息：保存 user / assistant 消息。
- Dify conversation_id：保存并用于连续对话。
- 更新时间：用于历史排序。

## 8. Dify 接入

环境变量：

```env
DIFY_API_KEY=app-your-dify-api-key
DIFY_API_URL=https://api.dify.ai/v1/chat-messages
```

请求方式：

- 前端调用本地 `/api/chat`。
- 本地接口使用服务端环境变量调用 Dify。
- API Key 不暴露到浏览器。
- 默认使用 `response_mode: streaming`。

## 10. 非功能要求

- 响应式：手机、平板、桌面均可使用。
- 可访问性：按钮含 aria-label，语义结构清晰。
- 视觉：毛玻璃、低饱和背景、清晰焦点、深浅色模式。
- 安全：Dify API Key 仅在服务端读取。
- 降级：未配置 Dify Key 时给出清晰提示，不导致页面崩溃。
